# envpact Shared Specification

**This is the single source of truth for all envpact components.**
Every component MUST follow these rules exactly so the ecosystem
interoperates without drift.

---

## 1. The Vault: `secrets.json`

A single JSON file in the user's private GitHub repo
`{username}/envpact-secrets`. Cloned locally to
`~/.envpact/secrets/`.

### Schema (v2 — supports environments)

```json
{
  "$schema": "https://envpact.oriz.in/schema/v2.json",
  "version": 2,
  "shared": {
    "OPENAI_API_KEY": "sk-...",
    "STRIPE_SECRET_KEY": "sk_live_..."
  },
  "projects": {
    "my-app": {
      "_default_env": "production",
      "OPENAI_API_KEY": "shared.OPENAI_API_KEY",
      "PORT": "3000",
      "DATABASE_URL": {
        "development": "postgres://localhost/myapp_dev",
        "staging": "shared.DATABASE_URL_STAGING",
        "production": "shared.DATABASE_URL_PROD"
      }
    }
  },
  "metadata": {
    "created_at": "2026-06-15T00:00:00Z",
    "updated_at": "2026-06-15T00:00:00Z",
    "owner": "chirag127"
  }
}
```

### Resolution Algorithm (canonical — every component uses this exact logic)

Given inputs `(secrets_json, project_name, environment)`:

1. Look up `project = secrets_json.projects[project_name]`. If
   missing, return empty dict (caller decides whether to prompt).
2. Determine effective environment:
   - If `environment` is explicitly provided, use it.
   - Else if `project._default_env` is set, use that.
   - Else use `"default"` (which means: pick scalar values directly).
3. For each `(key, value)` in `project` where `key != "_default_env"`:
   1. If `value` is a **string**:
      - If it starts with `shared.` (literal prefix, case-sensitive),
        look up `secrets_json.shared[value[7:]]`. If shared key
        missing, mark as UNRESOLVED and continue.
      - Else use `value` as-is.
   2. If `value` is an **object**:
      - Look up `value[environment]`. If missing, fall back to
        `value["default"]` if present, else mark UNRESOLVED.
      - The looked-up value is itself a string subject to the
        `shared.` prefix rule above (one level of indirection only,
        no recursion).
   3. If `value` is anything else: mark INVALID.
4. Return dict of `(key -> resolved_value)` plus list of
   UNRESOLVED keys (for caller prompting).

### Encryption (opt-in)

If a string value is wrapped in `enc:` prefix (e.g.
`"enc:age1xyz...ciphertext"`), it MUST be decrypted using the
user's age key at `~/.envpact/age.key` before resolution. If the
key is absent, surface a clear error. Encryption is opt-in per
secret — never auto-encrypt.

## 2. Local Configuration

| Path | Purpose |
| :--- | :--- |
| `~/.envpact/` | Root config directory |
| `~/.envpact/config.json` | Local config: `{ vault_repo, vault_url, last_sync, default_environment }` |
| `~/.envpact/secrets/` | Cloned vault repo (working tree) |
| `~/.envpact/secrets/secrets.json` | The vault file |
| `~/.envpact/age.key` | Optional age private key (mode 0600) |

`HOME` resolution (Windows compatible):
`process.env.USERPROFILE || process.env.HOME || os.homedir()`

`config.json` schema:
```json
{
  "version": 1,
  "vault_repo": "chirag127/envpact-secrets",
  "vault_url": "https://github.com/chirag127/envpact-secrets.git",
  "last_sync": "2026-06-15T12:34:56Z",
  "default_environment": "development",
  "auth_method": "auto"
}
```

## 3. Git Auth Detection (auto)

In order of preference:

1. **gh CLI**: if `gh auth status` exits 0, use
   `gh repo clone OWNER/REPO PATH` and `gh auth git-credential` for
   subsequent operations. Allows HTTPS without a stored PAT.
2. **SSH**: if `vault_url` is `git@github.com:...` or
   `~/.ssh/id_ed25519` exists, use `git clone git@github.com:...`.
3. **HTTPS PAT**: if `GITHUB_TOKEN` env var is set, use
   `https://oauth2:$GITHUB_TOKEN@github.com/...`.
4. **Fail with actionable error** listing all three options.

---

## 4. Project Auto-Detection

```
1. Try `git config --get remote.origin.url` in cwd.
2. If output matches /[:/]([^/]+)\/([^/]+?)(?:\.git)?$/,
   project name = capture group 2.
3. Else fallback to path.basename(cwd).
4. Always lowercase the result.
```

---

## 5. .env File Generation Rules

- Output path: `.env` (configurable via `--output`).
- Format: `KEY=VALUE\n` per line, no quoting unless value contains
  whitespace, `\n`, `\r`, `=`, `#`, or starts with whitespace —
  then double-quote and JSON-escape backslashes/quotes/newlines.
- Preserve key order from `.env.example` (if present).
- Add header comment: `# Generated by envpact on <ISO timestamp>\n# DO NOT COMMIT — add .env to .gitignore\n`.
- Append `.env` to `.gitignore` if not already present.

## 6. CLI Flags (CANONICAL — every CLI uses identical names)

```
envpact [options]

  --init [<git-url>|auto]   Initialize vault. "auto" creates a new
                              private repo via `gh repo create`.
  --github, -g               Sync resolved secrets to GitHub Actions
                              via `gh secret set`.
  --project <name>           Override auto-detected project name.
  --env <name>               Environment (development/staging/production).
                              Default: project._default_env or "default".
  --env-file <path>          Path to .env.example (default: .env.example).
  --output <path>            Output .env path (default: .env).
  --dry-run                  Print resolved env, do not write.
  --rotate <key>             Rotate a shared secret interactively.
  --list                     List all projects in vault.
  --list-shared              List shared secret names (values masked).
  --add <key>=<value>        Add/update a project secret.
  --add-shared <k>=<v>       Add/update a shared secret.
  --encrypt <key>            Encrypt a shared secret with age (in place).
  --decrypt <key>            Decrypt a shared secret with age (in place).
  --vault-pull               Pull latest vault state.
  --vault-push               Push pending vault changes.
  --version, -v
  --help, -h
```

## 7. MCP Tools (CANONICAL — both stdio and remote MCP expose these)

1. `generate_env(project_name?, environment?, working_directory?)` → writes .env, returns count + path.
2. `list_projects()` → array of project names.
3. `list_shared()` → array of shared secret names (values masked).
4. `add_secret(project_name, key, value, environment?)` → ok/error.
5. `add_shared_secret(key, value)` → ok/error.
6. `rotate_secret(key, new_value, sync_github?)` → list of affected projects.
7. `sync_github(project_name?, environment?)` → count synced.
8. `list_environments(project_name)` → array of env names available for project.

All tool descriptions and JSON schemas are in
`_build/specs/MCP_TOOLS.json`.

## 10. Conventions Every Component Follows

- **Zero runtime dependencies** where possible (Node CLI: stdlib
  only; Python: stdlib only; MCP server may use
  `@modelcontextprotocol/sdk` only).
- **Cross-platform paths**: always `path.join(...)` /
  `pathlib.Path(...)`.
- **Never log secret values**: not in errors, not in `--list-shared`
  (mask with `****`), not in MCP responses (mask in
  `list_shared`/`list_environments`).
- **Validate schema before use**: reject `secrets.json` with
  unknown `version` field.
- **Atomic writes**: write to `.env.tmp` then `rename()` to `.env`.
- **All commits to vault are signed-off**: include `Signed-off-by:`
  trailer in commit messages from CLI.
- **Test coverage target**: ≥80% line coverage for resolver,
  parser, and vault modules.
